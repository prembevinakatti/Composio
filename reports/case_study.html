How the Research Agent Works

Step 1: Load Targets

Load application list from initial seed CSV file.

Step 2: Deep Web Search

Agent issues precise queries via Tavily to locate official developer documentation and API references.

Step 3: Structured Extraction

Groq LLaMA-3 processes HTML context and maps unstructured text into a strict Pydantic JSON schema.

Step 4: Classification

Agent categorizes authentication logic (OAuth2, API Key, Basic) and API surface size.

Step 5: Verification

A distinct Verification Agent reviews the output, fact-checking for hallucinations against rules.

Step 6: Report Generation

Data is aggregated, passed through an Insights engine, and compiled into this static HTML artifact.

Lessons Learned

1. AI is fast, but hallucinates

Without strict Pydantic schemas and prompt engineering, LLMs will invent API endpoints if documentation is sparse.

2. Verification Loops are Critical

Implementing a second 'Verifier' agent boosted accuracy from 76% to 93% by catching logical inconsistencies.

3. Official Docs > Generic Web Search

Forcing the agent to append "developer documentation" to Tavily queries drastically reduced noise.

4. Human Review Remains Necessary

For the last 7% of edge cases (mostly partner-gated APIs), human intuition is still required.
